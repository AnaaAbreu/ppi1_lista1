HTTP

Papel do HTTP na comunicação entre navegador e servidor

O HTTP (HyperText Transfer Protocol) é o protocolo responsável pela comunicação entre o navegador e o servidor Web. Sempre que um usuário acessa um site, o navegador envia uma requisição HTTP ao servidor, que responde enviando os recursos solicitados, como páginas HTML, imagens e outros arquivos.


Cliente, servidor, requisição e resposta

O cliente é o navegador utilizado pelo usuário para acessar um site. O servidor é o computador que armazena os arquivos da aplicação Web. A requisição é a mensagem enviada pelo cliente solicitando um recurso ao servidor. A resposta é a mensagem enviada pelo servidor contendo o recurso solicitado ou um código indicando o resultado da solicitação.


Métodos HTTP

GET

O método GET é utilizado para solicitar informações ao servidor, como abrir uma página ou visualizar um produto.

POST

O método POST é utilizado para enviar informações ao servidor, como dados de um formulário de cadastro ou login.


Exemplo de Requisição HTTP

 GET /index.html HTTP/1.1 
        Host: www.exemplo.com 
        User-Agent: Mozilla/5.0

Exemplo de Resposta HTTP

HTTP/1.1 200 OK 
        Content-Type: text/html
        <html>
         ... 
         </html>

Códigos de Status HTTP

  • 200

    Requisição realizada com sucesso.

  • 301

    O recurso foi movido permanentemente para outro endereço.

  • 400

    A requisição enviada pelo cliente é inválida.

  • 403

    O acesso ao recurso foi proibido.

  • 404

    O recurso solicitado não foi encontrado.

  • 500

    Erro interno no servidor.
